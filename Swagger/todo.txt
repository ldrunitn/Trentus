dividere api per oggetto trattato (fare unfile per servizi, utenti, gds, segnalazioni, avvisi, sondaggi)

gli errori li gestiamo come content in un altro file
gli oggetti li gestiamo come content nello stesso file
i parametri riguardano il path e la query (anche gli header ma non mi sembra li abbiamo usati)


Formato del path delle nostre api (esempio):

    get:
      summary: 
      description: 
      security: (optional)
        - BearerAuth: 
      parameters: (optional) 
      responses:
	'200':
	  description:
	    content: 
	      $ref: '#/components/object'
	'401': (optional)
          $ref: '../errors.yaml#/components/Unauthorized'

      description:

          $ref: '../errors.yaml#/components/Unauthorized'
